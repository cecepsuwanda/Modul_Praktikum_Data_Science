\SesiRPS{14}{P10}{$\sim$170 menit}{CPMK-P4}

\noindent Peer review meniru praktik tim data di industri: kode dan narasi diperiksa oleh rekan sebelum digabungkan ke repositori bersama. Tujuannya bukan mencari kesalahan untuk menilai, melainkan meningkatkan kejelasan dan menemukan dependensi tersembunyi.

\section{Kemampuan akhir sesi}

Menjalankan notebook orang lain dari awal; mendokumentasikan kegagalan reproduksi secara spesifik; merevisi notebook sendiri berdasarkan umpan balik; menyertakan daftar paket versi.

Reproduktifitas bersifat sosial-teknis: versi paket saja tidak cukup jika urutan sel atau asumsi path tidak didokumentasikan. Catat langkah manual (unduh data, set variabel lingkungan) di Markdown.

\section{Teori}

\begin{konsep}
Reproduktifitas membutuhkan versi Python/paket, urutan sel, dan akses data yang sama. \textbf{Peer review} meningkatkan kejelasan dan menemukan langkah yang ``hanya jalan di laptop pemilik''.
\end{konsep}

\section{Prosedur}

Prosedur berikut memastikan umpan balik dapat ditindaklanjuti: setiap poin review sebaiknya merujuk nomor sel atau judul Markdown agar mitra mudah memperbaiki.

\begin{enumerate}[leftmargin=*]
  \item Tukar notebook dengan pasangan; \textbf{Restart \& Run All}.
  \item Catat sel gagal, missing file, instruksi kurang.
  \item Isi lembar review (lihat Lampiran): minimal dua saran spesifik.
  \item Revisi notebook menjadi \texttt{\ldots v2.ipynb}; lampirkan \texttt{requirements.txt}.
\end{enumerate}
