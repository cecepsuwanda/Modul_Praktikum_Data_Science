\section{Teori dan bahan kajian}

Notebook interaktif menggabungkan narasi dan eksperimen dalam satu berkas yang dapat dibagikan. Memahami perbedaan sel, kernel, dan urutan eksekusi mencegah kesalahan yang sering muncul saat demo atau pengumpulan tugas.

\subsection{Sel, kernel, dan urutan eksekusi}

\begin{konsep}
\jupyter{} menggabungkan narasi (\textbf{sel Markdown}) dan kode (\textbf{sel kode}). \textbf{Kernel} adalah proses Python yang menjalankan kode. Urutan angka di sisi sel menunjukkan urutan eksekusi terakhir---bukan urutan di halaman. Setelah mengubah banyak sel, gunakan \textbf{Restart Kernel and Run All} untuk memverifikasi reproduktifitas.
\end{konsep}

Kernel yang tidak sinkron dengan kode di layar adalah sumber bug klasik: variabel lama tetap ada di memori meskipun sel sudah dihapus dari dokumen. Kebiasaan \textit{Run All} setelah penyuntingan besar mengurangi risiko ini dan menyiapkan Anda untuk lingkungan penilaian otomatis.

\subsection{CRISP-DM dalam satu halaman}

\textit{Cross-Industry Standard Process for Data Mining} memetakan proyek data ke enam fase: \textbf{Business Understanding} (tujuan bisnis), \textbf{Data Understanding} (koleksi, kualitas), \textbf{Data Preparation} (cleaning, fitur), \textbf{Modeling}, \textbf{Evaluation}, \textbf{Deployment}. Pada praktikum 1 SKS, fokus kita adalah memahami bahwa notebook yang baik mencerminkan alur ini meskipun tidak semua fase dieksekusi penuh setiap minggu.

\subsection{Etika dan lisensi data}

\begin{catatan}
Cantumkan \textbf{URL sumber}, \textbf{lisensi} (CC, domain publik, syarat Kaggle, dll.), dan hindari mengunggah data yang berisi \textbf{PII} (data pribadi) tanpa persetujuan. Jika memakai data sintetis, tulis secara eksplisit bahwa data sintetis.
\end{catatan}

Dokumentasi lisensi bukan formalitas administratif: ia melindungi Anda dan institusi dari pelanggaran syarat penggunaan data. Di dunia profesional, pelanggaran lisensi dapat mengakibatkan penarikan model dari produksi.

\section{Contoh kode dan studi kasus}

Contoh berikut dapat disalin per blok ke notebook; sesuaikan string identitas dan URL dataset dengan data Anda. Setelah setiap blok, tambahkan satu paragraf Markdown yang menjelaskan hubungan blok tersebut dengan fase CRISP-DM.

\subsection{Memeriksa versi Python di notebook}

\MulaiPenjelasanKode Blok berikut memverifikasi bahwa kernel notebook memakai interpreter Python yang Anda duga. Modul \texttt{sys} menyediakan \texttt{sys.version} (string versi) dan \texttt{sys.executable} (path berkas \texttt{python}); keduanya sering dipaketkan dalam sel pertama saat debugging lingkungan atau saat dokumentasi tugas. \SelesaiPenjelasanKode

\begin{lstlisting}[caption=Versi Python dan path executable]
import sys  # modul standar: informasi interpreter

print(sys.version)  # string versi Python (major.minor.patch + build)
print(sys.executable)  # path absolut ke berkas python yang menjalankan kernel
\end{lstlisting}

\subsection{Markdown vs kode}

\MulaiPenjelasanKode Contoh ini membedakan \textbf{komentar Python} (baris dengan \texttt{\#}, tidak dieksekusi sebagai perintah) dari \textbf{teks narasi} yang ditulis di sel Markdown. Variabel \texttt{nama\_kampus} bertipe \texttt{str}, \texttt{semester} bertipe \texttt{int}; \texttt{f"..."} adalah \textit{f-string} untuk menyisipkan nilai variabel ke dalam satu string cetak. \SelesaiPenjelasanKode

\begin{lstlisting}[caption=Operasi sederhana di sel kode]
# Judul sel: perkenalan (komentar Python - penjelasan untuk pembaca kode)
nama_kampus = "Universitas Bale Bandung"  # str: contoh literal teks
semester = 2026  # int: label periode (boleh disesuaikan)
print(f"Praktikum Data Science | {nama_kampus} | {semester}")  # f-string: interpolasi {nama_kampus} dan {semester}
\end{lstlisting}

\subsection{Struktur dictionary untuk CRISP-DM ringkas}

\MulaiPenjelasanKode \texttt{dict} memetakan kunci (nama fase CRISP-DM) ke nilai (catatan singkat proyek). Metode \texttt{.items()} menghasilkan pasangan \texttt{(kunci, nilai)} untuk diiterasi; pola \texttt{for k, v in dict.items()} adalah cara standar mencetak ringkasan dokumentasi di notebook tanpa pustaka tambahan. \SelesaiPenjelasanKode

\begin{lstlisting}[caption=Mapping fase CRISP-DM ke catatan proyek mini]
crisp_dm = {  # dict: kunci str -> nilai str (satu kalimat per fase)
    "business_understanding": "Mengurangi churn pelanggan e-commerce",
    "data_understanding": "Dataset transaksi terbuka, cek missing",
    "data_prep": "Imputasi, encoding kategori",
    "modeling": "Klasifikasi churn dengan RandomForest",
    "evaluation": "F1-score, confusion matrix",
    "deployment": "Dashboard internal (di luar cakupan praktikum)",
}
for fase, catatan in crisp_dm.items():  # .items(): iterasi pasangan (kunci, nilai)
    print(f"{fase}: {catatan}")  # satu baris cetak per fase
\end{lstlisting}

\subsection{Menyimpan metadata sumber data (etika)}

\MulaiPenjelasanKode Kamus \texttt{Sumber\_data} menyimpan metadata etika (URL, lisensi, tanggal, catatan) dalam struktur yang mudah disalin ke laporan. \texttt{json.dumps} mengubah kamus menjadi teks JSON; argumen \texttt{indent=2} merapikan baris, \texttt{ensure\_ascii=False} mempertahankan aksara non-ASCII (mis.\ bahasa Indonesia). \SelesaiPenjelasanKode

\begin{lstlisting}[caption=Blok dokumentasi sumber data di notebook]
Sumber_data = {  # ganti URL/lisensi sesuai dataset Anda
    "url": "https://www.kaggle.com/datasets/...",
    "lisensi": "CC0 / sesuai halaman dataset",
    "tanggal_unduh": "2026-04-10",
    "catatan": "Tidak untuk komersial tanpa izin pemilik dataset.",
}
import json  # modul standar: serialisasi JSON

print(json.dumps(Sumber_data, indent=2, ensure_ascii=False))  # pretty-print; ensure_ascii=False: Unicode tidak di-escape
\end{lstlisting}
